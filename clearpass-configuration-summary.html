A Complete Guide to ClearPass AAA Configuration

I use clearpass policy manager. Pls guide me how to config clearpass as AAA for cisco device via tacacs

Guide 1: ClearPass for Cisco Devices (TACACS+)

This was the first part of our discussion, focused on setting up ClearPass to manage administrative access to Cisco network devices using the TACACS+ protocol.

Step 1: Add Cisco Device to ClearPass

Define the Cisco device in ClearPass under Configuration > Network > Devices. Set the Vendor Name to Cisco and configure a shared secret.

Step 2: Create Enforcement Profiles

These profiles define the permissions. For Cisco, we use the priv-lvl attribute.

  • Read-Only Profile: Set attribute priv-lvl = 1.
  • Admin Profile: Set attribute priv-lvl = 15.

Step 3: Define Enforcement Policy

Create a policy at Configuration > Enforcement > Policies to map authentication results (like AD group membership) to the enforcement profiles.

Step 4: Create the TACACS+ Service

The service listens for requests. Create a TACACS+ Enforcement service at Configuration > Services that matches requests from your Cisco device's IP.

Step 5: Configure the Cisco Device (CLI)

On the switch, configure AAA to point to ClearPass:

aaa new-model
!
tacacs server CLEARPASS_SERVER_IP
 key YOUR_SHARED_SECRET
!
aaa group server tacacs+ ClearPass_Group
 server-name CLEARPASS_SERVER_IP
!
aaa authentication login default group ClearPass_Group local
aaa authorization exec default group ClearPass_Group local
aaa authorization commands 15 default group ClearPass_Group local
!
ip tacacs source-interface Vlan10

Guide 2: ClearPass for Aruba IAP (TACACS+)

Next, we expanded the configuration to cover management access for Aruba Instant Access Point (IAP) clusters. The key difference is the use of an Aruba-specific attribute.

Core Concept: The `Aruba-Admin-Role` Attribute

Instead of `priv-lvl`, Aruba uses the VSA Aruba-Admin-Role. The value of this attribute must match the name of a management role defined on the IAP itself.

Step 1: Define Management Roles on IAP

In the IAP's web UI, under System > Admin > Roles, create roles like read-only and administrator.

Step 2: Add IAP to ClearPass

Add the IAP's virtual controller IP to ClearPass under Network Devices. Set the Vendor Name to Aruba.

Step 3: Create Enforcement Profiles

  • Read-Only Profile: Set attribute Aruba-Admin-Role = read-only.
  • Admin Profile: Set attribute Aruba-Admin-Role = administrator.

Step 4: Configure IAP for TACACS+

In the IAP's UI, go to System > Admin, select Authentication Server, and add your ClearPass server details and shared key. It's best practice to set the 'Local' server as a fallback.

Guide 3: ClearPass for Check Point Gaia (TACACS+)

We then added Check Point firewalls to our centralized AAA strategy, again using a vendor-specific attribute for role mapping.

Core Concept: The `checkpoint-role` Attribute

Check Point uses the checkpoint-role attribute to assign administrators to roles defined in the Gaia OS.

Step 1: Note Check Point Roles

In the Gaia Portal, under User Management > Roles, identify the roles you want to use, such as adminRole and readOnlyRole.

Step 2: Add Firewall to ClearPass

Add the firewall's management IP to Network Devices in ClearPass. Set the Vendor Name to CheckPoint.

Step 3: Create Enforcement Profiles

  • Admin Profile: Set attribute checkpoint-role = adminRole.
  • Read-Only Profile: Set attribute checkpoint-role = readOnlyRole.

Step 4: Configure TACACS+ on Gaia

In the Gaia Portal, go to User Management > Authentication Servers to add your ClearPass server and shared key. Then, assign this server to an authentication profile.
