DC/DR Network Architecture Proposal

For Vietnam Operations

Executive Summary

This proposal outlines three strategic options for establishing a primary Data Center (DC) and Disaster Recovery (DR) solution for your operations in Vietnam. The objective is to significantly enhance resilience, security, and scalability. Each option includes a detailed analysis of routing paths for accessing key resources.

  • Option 1: Centralized Production DC with Active-Passive DR (at Office): A robust, cost-effective model using a new colocation DC for production and an existing office as the standby DR site.
  • Option 2: Geo-Clustered Active-Active DC: An advanced, high-availability model where two sites actively serve traffic simultaneously, offering minimal downtime at a higher cost and complexity.
  • Option 3: Dual Colocation Active-Passive DC/DR: The gold-standard approach, establishing both primary DC and DR sites in separate, purpose-built colocation facilities for maximum resilience.

Option 1: Centralized Production DC with Active-Passive DR (at Office)

1. High-Level Strategy

This design designates the new colocation data center, VN-DCH-01, as the single, primary production site. VN-OFFICE-01 will be re-purposed as the warm-standby Disaster Recovery (DR) site.

2. Network Architecture Diagram

+--------------------------+ | Global Corporate HQ | +--------------------------+ | | (Global P2P Link 1) --------------------' '-------------------- (Global P2P Link 2) | | v v +---------------------------------------------------------------------------------------------------+ | NEW PRIMARY DC (VN-DCH-01) | | (Production) | | +----------------------+ +----------------------+ +----------------------+ +----------------+ | | | Internet Link #1 | | Internet Link #2 | | Global P2P Links | | DCI | | | | (Primary ISP) | | (Secondary ISP) | | (Terminates Here) | | (Dark Fiber/L2)| | | +-------+--------------+ +--------+-------------+ +---------+------------+ +-------+--------+ | | | | | | | | v v v | | | [SD-WAN/Router] <--> [NGFW Cluster] <--> [Core Switch Cluster] <--> [Server/SAN] | | | ^ | | +--------------------------------|-----------------------------------------------------|----------+ | (SD-WAN over Internet/MPLS) | (Data Replication) | | +--------------------------------|-----------------------------------------------------|----------+ | ^ v v | | | [SD-WAN/Router] <--> [NGFW Cluster] <--> [Core Switch Cluster] <--> [Replicated SAN] | | | | | | DR SITE (VN-OFFICE-01) | | | (Passive) | | +----------------------------------------------------------------------------------------------+ | | | | (User Traffic from VN-OFFICE-01 Access Layer) | | +----|---------------------------------------------------------------------------------------------+ | (SD-WAN over Internet) | +----------------------> [VN-OFFICE-02] -----> [Branch 3, 4, 5] (Internet link here acts as backup path for all sites)

3. Detailed Routing and Traffic Paths

  • Accessing Internal Vietnam Apps:

    Path: User at Branch -> SD-WAN -> Tunnel over Internet -> SD-WAN Headend at VN-DCH-01 -> NGFW -> Core -> App Server.

  • Accessing Global HQ Apps:

    Path: User at Branch -> SD-WAN -> Tunnel over Internet -> SD-WAN Headend at VN-DCH-01 -> NGFW -> Core -> Global P2P Link -> Global HQ.

  • Accessing Public Internet:

    Path: User at Branch -> SD-WAN -> Tunnel over Internet -> SD-WAN Headend at VN-DCH-01 -> NGFW -> Internet Links.

4. Recovery Objectives

RTO: 2-4 hours
RPO: < 15 minutes

Option 2: Geo-Clustered Active-Active DC

1. High-Level Strategy

This advanced design treats both the new colocation VN-DCH-01 and the primary office VN-OFFICE-01 as two independent, active production data centers, load-balancing traffic between them.

2. Network Architecture Diagram

+--------------------------+ | Global Corporate HQ | +--------------------------+ | | (Global P2P Link 1) --------------------' '-------------------- (Global P2P Link 2) | | v v +-----------------------------+ +--------------------------------+ | ACTIVE DC 1 (VN-DCH-01) | | ACTIVE DC 2 (VN-OFFICE-01) | | +---------+ +--------+ | +-------------------+ | +---------+ +--------+ | | |Internet | |Global | | | GSLB / | | |Internet | |Global | | | |Link #1 | |P2P #1 | |<-------> | Anycast Routing | <------->| |Link #3 | |P2P #2 | | | +----+----+ +---+----+ | +-------------------+ | +----+----+ +---+----+ | | | | | ^ | | | | | v v | | (User Traffic) | v v | | [SD-WAN/Rtr]<->[NGFW]<->[Core]<->[App/SAN] | | [SD-WAN/Rtr]<->[NGFW]<->[Core]<->[App/SAN] | | | | | | | ^ +-----------------|-------------------+ ^ | | | | | | | '--------------------------------------|-----------------------------------------' | | (High-Speed DCI for Synchronous Replication) | | | | +-------------------------------------------------|----------------------------------------------------+ | (SD-WAN over Internet) | v [VN-OFFICE-02] -----> [Branch 3, 4, 5]

3. Detailed Routing and Traffic Paths

  • Accessing Internal Vietnam Apps:

    Path: GSLB/SD-WAN directs user to the best DC (VN-DCH-01 or VN-OFFICE-01). Traffic is served locally at that DC.

  • Accessing Global HQ Apps:

    Path: SD-WAN directs traffic to the closest DC with an active P2P link (P2P #1 at DC1, P2P #2 at DC2).

  • Accessing Public Internet:

    Path: SD-WAN directs traffic to the nearest DC for local internet breakout.

4. Recovery Objectives

RTO: Near-zero
RPO: Zero

Option 3: Dual Colocation Active-Passive DC/DR (Gold Standard)

1. High-Level Strategy

This design establishes a primary production DC in VN-DCH-01 and a dedicated DR site in a second, geographically separate colocation facility, VN-DCH-02. This offers maximum resilience.

2. Network Architecture Diagram

+--------------------------+ | Global Corporate HQ | +--------------------------+ | | (Global P2P Link 1) --------------------' '-------------------- (Global P2P Link 2) | | v v +-----------------------------+ +--------------------------------+ | PRIMARY DC (VN-DCH-01) | (High-Speed DCI Link) | DR SITE (VN-DCH-02) | | (Active) |<------------------------------------->| (Passive) | | +---------+ +--------+ | (For Data Replication) | +---------+ +--------+ | | |Internet | |Global | | | |Internet | |Global | | | |Link #1 | |P2P #1 | | | |Link #2 | |P2P #2 | | | +----+----+ +---+----+ | | +----+----+ +---+----+ | | | | | | | (Standby) | (Standby)| | v v | | v v | | [SD-WAN/Rtr]<->[NGFW]<->[Core]<->[Prod SAN] | [SD-WAN/Rtr]<->[NGFW]<->[Core]<->[Replica SAN] | | | | | +-------------------------------+---------------------------------------+--------------------------------+ | | (SD-WAN over Internet) v [VN-OFFICE-01/02] -----> [Branch 3, 4, 5] (Now standard branch/office sites)

3. Detailed Routing and Traffic Paths

Routing paths are identical to Option 1 in normal operation. All traffic is directed to the primary DC (VN-DCH-01). The key difference is the DR site (VN-DCH-02) is a hardened facility with its own standby global and internet links, providing superior failover capabilities.

4. Recovery Objectives

RTO: 1-3 hours
RPO: < 15 minutes

Recommendation

For a global company establishing its first formal DC/DR architecture in Vietnam, I strongly recommend Option 1: Centralized Production DC with Active-Passive DR (at Office).

Justification:

  • Pragmatic Step-Up: It provides a massive leap in resilience and maturity over your current state without incurring the cost and complexity of the other two options.
  • Cost/Benefit Balance: It delivers the best "bang for the buck," achieving a robust DR posture that meets the needs of most businesses.

However, if your company's risk tolerance is extremely low and the budget allows, Option 3: Dual Colocation Active-Passive DC/DR is the technically superior and most resilient long-term solution. It represents the industry's best practice and should be considered your strategic target architecture.
